carpal Futures overview

Overview

The central class of the carpal futures mechanism is the template class carpal::PromiseFuturePair<T>. This is an object that will receive the return value of an asynchronous operation upon its completion. It may also receive an exception, if the asynchronous operation throws.

carpal::PromiseFuturePair<T> allows to:

  • check whether the operation has completed or not and, if completed, whether it returned a value or threw an exception;
  • retrieve the returned value or the exception, if completed;
  • wait for completion;
  • enqueue an operation — called continuation — to be executed when the asynchronous operation completes;
  • declare the asynchronous operation complete by specifying the returned value or the exception.

The consumer — the caller of the asynchronous operation — typically receive a carpal::Future<T>, which is essentially a wrapper over a std::shared_ptr<PromiseFuturePair<T> >. A carpal::Future<T> offers access to the consumer facing operations of the carpal::PromiseFuturePair, plus some methods allowing to compose asynchronous operations.

Some of those methods involve executing code on an executor mechanism.

On the other hand, carpal::Promise<T> is, similarly, a wrapper over a std::shared_ptr<PromiseFuturePair<T> > giving access to its producer facing methods.

There are also stand-alone functions related to futures.
